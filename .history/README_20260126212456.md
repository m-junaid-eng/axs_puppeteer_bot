# Puppeteer opener (compliant)

This project opens an AXS URL using Puppeteer, saves HTML/screenshots, and (optionally) follows "See Event" links to collect event detail pages.

It does **not** attempt to bypass CAPTCHAs or evade Cloudflare/Turnstile. In `--interactive` mode it pauses so you can solve challenges manually.

## Install

```bash
npm install
```

## Run

```bash
npm run open:axs
```

## Common commands

- Open a custom URL:
	```bash
	npm run open:axs -- --url "https://example.com"
	```

- Headless mode:
	```bash
	npm run open:axs -- --headless
	```

- Save page HTML (default on) / disable:
	```bash
	npm run open:axs -- --no-save-html
	```

## All Events scraping

Follow "See Event" links and save each event HTML/PNG:

```bash
npm run open:axs -- --all-events --max-events 10 --delay-ms 1500
```

### Interactive mode (manual solve)

Run with a visible browser window. If Cloudflare/Turnstile appears, solve it manually in the browser and then press Enter in the terminal.

```bash
npm run open:axs -- --all-events --interactive --max-events 5
```

Helpful pacing flags (can reduce repeat prompts):

```bash
# Step-by-step: press Enter before each event navigation
npm run open:axs -- --all-events --interactive --step

# Extra wait after you solve a challenge (default in interactive is 6000ms)
npm run open:axs -- --all-events --interactive --post-solve-wait-ms 12000
```

Note: If Cloudflare challenges keep appearing for every event, that is often driven by account/IP/network reputation and may require manual solving per event.

## Profiles

By default, `--interactive` uses a persistent profile at `.state/profiles/interactive` so cookies/session state can carry across runs.

- `--fresh-profile`: start with a clean, new profile directory each run (more likely to trigger challenges)
- `--profile <dir>`: use a specific profile directory (relative to project root, or absolute)

Examples:

```bash
npm run open:axs -- --all-events --interactive --profile .state/profiles/my-profile
npm run open:axs -- --all-events --interactive --fresh-profile
```

## User-Agent options

- In `--interactive` mode, the script defaults to **not overriding** the browser UA.
- Use `--ua "..."` to set a specific UA.
- Use `--rotate-ua` to force UA rotation from `uas.json`.
- Use `--no-ua-override` to prevent UA override.

## Use installed Chrome (recommended for fewer challenges)

Cloudflare often challenges automated/bundled Chromium more aggressively. Without doing any stealth/bypass, you can try running with your installed Google Chrome:

```bash
npm run open:axs -- --all-events --interactive --use-system-chrome
```

Or specify an explicit Chrome path:

```bash
npm run open:axs -- --all-events --interactive --chrome-path "C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe"
```

## Output

Artifacts are written under `artifacts/`:

- `artifacts/axs.html`, `artifacts/axs.png`
- `artifacts/all-events.html`
- `artifacts/events/*.html` and `artifacts/events/*.png`
- `artifacts/events.json` and `artifacts/events.xlsx` (when `--all-events` and XLSX enabled)

Fingerprint signal snapshots (for debugging/learning only) are saved to `artifacts/fingerprint.json` and `artifacts/fingerprints/` (disable with `--no-fingerprint`).
