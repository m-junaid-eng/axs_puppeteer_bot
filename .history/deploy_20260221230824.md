# Hetzner deployment (HEADFUL) with Xvfb + cron

This project runs a real Chrome browser with Puppeteer. On a Hetzner VPS (no physical screen), you can still run it **headfully** by using **Xvfb** (a virtual display) and scheduling with **cron**.

> Note
>
>- This guide is **HEADFUL** (no `--headless`).
>- The current script launches the browser with `headless: false` by default, so Xvfb is the correct way to run it on a server.
>- If AXS shows CAPTCHA/challenges, cron cannot solve them.
>- At the moment, AXS often auto-checks and verifies the challenge (no manual checkbox click needed).
---

## 1) Create a server user (recommended)

```bash
sudo adduser --disabled-password --gecos "" axs

# Set a password for `axs` (required for sudo after you switch to this user)
# Note: sudo always asks for the *current user's* password (not root, not the previous user).
sudo passwd axs

sudo usermod -aG sudo axs
sudo -iu axs
```

sudo apt-get update

## 3) Install Node.js (LTS)

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v
npm -v
```

---

## 4) Install Google Chrome

Recommended (Google Chrome Stable):

```bash
curl -fsSL https://dl.google.com/linux/linux_signing_key.pub \
  | sudo gpg --dearmor -o /usr/share/keyrings/google-linux.gpg

echo "deb [arch=amd64 signed-by=/usr/share/keyrings/google-linux.gpg] http://dl.google.com/linux/chrome/deb/ stable main" \
  | sudo tee /etc/apt/sources.list.d/google-chrome.list > /dev/null

sudo apt-get update
sudo apt-get install -y google-chrome-stable

google-chrome --version
```

## 5) Deploy the project

```bash
sudo mkdir -p /opt/axs-bot
sudo chown -R axs:axs /opt/axs-bot

cd /opt
git clone <YOUR_REPO_URL> axs-bot
cd /opt/axs-bot

npm install
```

## 6) Create `.env`

```bash
cd /opt/axs-bot
nano .env
```

Example:

```env
DISCORD_TOKEN=YOUR_DISCORD_BOT_TOKEN
```

Lock it down:

```bash
chmod 600 /opt/axs-bot/.env
```

## 7) Supabase requirement

Your Supabase table `axs_links` must have the Discord channel id in:

- `category_channel_id`

If it’s missing, the script will skip Discord sending for that row.

---

## 8) Create a headful runner script (Xvfb)

Create `/opt/axs-bot/run-cron.sh`:

```bash
nano /opt/axs-bot/run-cron.sh
```

Paste:

```bash
#!/usr/bin/env bash
set -euo pipefail

cd /opt/axs-bot

# Cron has a smaller PATH; keep it explicit
export PATH="/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin"
export NODE_ENV=production

# HEADFUL run using a virtual display
xvfb-run -a --server-args="-screen 0 1920x1080x24" \
  /usr/bin/node scripts/open-axs.mjs --all-events
```

Make it executable:

```bash
chmod +x /opt/axs-bot/run-cron.sh
```

---

## 9) Test run manually

```bash
/opt/axs-bot/run-cron.sh
```

---

## 10) Add cron job

Edit crontab:

```bash
crontab -e
```

Example (every 10 minutes):

```cron
*/10 * * * * /opt/axs-bot/run-cron.sh >> /opt/axs-bot/cron.log 2>&1
```

---

## 11) Monitor logs

```bash
tail -n 200 -f /opt/axs-bot/cron.log
```

---

## 12) Common issues

### Cron can’t find node

Use full path (the script uses `/usr/bin/node`). Confirm:

```bash
which node
```

### CAPTCHA / challenge pages

Cron cannot solve challenges. If you need manual solving, run with `--interactive` in an SSH session (not in cron).
